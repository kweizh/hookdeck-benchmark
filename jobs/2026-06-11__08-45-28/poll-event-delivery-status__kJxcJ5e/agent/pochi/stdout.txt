## What it does

1. **Reads** `EVENT_ID` and `HOOKDECK_API_KEY` from the environment; exits 1 immediately if either is missing.
2. **Polls** `GET https://api.hookdeck.com/2025-07-01/events/{id}` with Bearer-token auth, counting each call as an attempt.
3. **Checks** the event's `status` field after every poll:
   - `SUCCESSFUL` â†’ emits output, exits **0**
   - `FAILED` â†’ emits output, exits **1**
   - Any non-terminal state (`QUEUED`, `SCHEDULED`, `HOLD`) â†’ sleeps and retries
4. **Exponential back-off**: starts at 0.25 s, doubles each time (0.25 â†’ 0.5 â†’ 1 â†’ 2 â†’ 4 â†’ 8 â†’ â€¦). Each sleep is capped at the remaining budget so the script never exceeds the 30-second limit.
5. **Timeout**: if the budget runs out before a terminal state is reached, emits output and exits **1**.
6. **Reads `response_status` directly** from the event object (Hookdeck surfaces the last HTTP response code as a top-level integer field; `null` when Hookdeck has not yet attempted delivery).

## Output format (always exactly two lines on stdout)

```
attempt_count=<int>
response_status=<int|none>
```
