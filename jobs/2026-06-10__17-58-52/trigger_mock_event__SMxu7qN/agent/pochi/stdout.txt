## What the script does

1. **Generates a unique `run-id`** (8-char hex from timestamp hash) so all resource names are distinct per run.

2. **Authenticates** the Hookdeck CLI using `hookdeck ci --api-key $HOOKDECK_API_KEY` for headless CI use.

3. **Creates a connection** named `mock-conn-<run-id>` linking:
   - Source: `mock-source-<run-id>` (type: WEBHOOK)
   - Destination: `mock-dest-<run-id>` (type: **MOCK_API** — the correct CLI type for Hookdeck's Mock destination)

4. **Publishes an event** by POST-ing JSON to the source's Hookdeck URL (`https://hkdk.events/<source_id>`).

5. **Retrieves the event ID** from the Events API (`https://api.hookdeck.com/2025-07-01/events?source_id=...`) with up to 12 retries.

6. **Appends** the event ID to `/home/user/hookdeck-task/output.log` in the exact format `Event ID: <event_id>`.
