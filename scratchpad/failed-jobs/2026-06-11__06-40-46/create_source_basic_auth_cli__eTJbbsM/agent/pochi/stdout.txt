## Technical note

The Hookdeck API requires the authentication config to be nested under `config.auth_type` and `config.auth` (not the top-level `verification` field which was deprecated). The CLI's `--basic-auth-user` and `--basic-auth-pass` flags also map to the wrong API fields for source creation, so the REST API was used directly via `POST /2025-07-01/sources` with the proper `config` payload.
